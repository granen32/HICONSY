# PRD

## Product Summary

HICONSY는 수학 문제를 폴더와 키워드 기준으로 탐색하고, 이미지 기반으로 등록할 수 있는 모바일 WebView 우선 서비스다. 현재 구현은 메인 탐색, 폴더 컨텍스트 탐색, 문제 상세, 문제 등록까지 포함한다.

## Core UX Model

### 1. 홈

- 사용자는 `/`에서 전체 문제를 본다.
- 상단에는 폴더 캐러셀이 있다.
- 폴더를 누르면 해당 폴더 컨텍스트로 이동한다.
- 키워드 영역은 그리드/리스트 토글을 제공한다.
- 첫 방문에는 튜토리얼 오버레이가 노출된다.

### 2. 폴더 컨텍스트

- `뮤직`은 기본 폴더 컨텍스트이며 `/music`으로 접근한다.
- 일반 폴더는 `/folders/[id]`로 접근한다.
- 폴더 컨텍스트에서는 해당 폴더 문제만 노출된다.
- 키워드 탭을 누르면 같은 폴더 안에서만 필터링된 피드로 이동한다.

### 3. 키워드 피드

- 메인 컨텍스트: `/keywords/[slug]`
- 뮤직 컨텍스트: `/music/keywords/[slug]`
- 일반 폴더 컨텍스트: `/folders/[id]/keywords/[slug]`
- 피드는 카드형 세로 피드 구조를 사용한다.
- 이미지가 여러 장이면 캐러셀처럼 좌우 스와이프 가능한 구조로 보인다.

### 4. 문제 상세

- 컨텍스트별 상세 라우트로 진입한다.
- 뒤로가기는 진입한 탐색 컨텍스트를 유지한다.
- 삭제 후에도 해당 컨텍스트로 복귀한다.

### 5. 문제 등록

- 대표 이미지, 제목, 키워드, 폴더, 해설 이미지, 본문 입력을 지원한다.
- 선택 이미지는 `objectUrl` 기반 preview로만 관리한다.
- 키워드 선택은 전용 패널로 처리한다.

## Seed Data Rules

- 기본 문제 수: 20개
- 기본 폴더:
  - `뮤직`
  - `기출`
  - `6월 모의`
- 각 폴더는 자체 문제 집합을 가진다.
- 홈은 전체 20개를 보여준다.

## Functional Requirements

### Listing

- 폴더 캐러셀 제공
- 폴더 진입 시 컨텍스트별 리스트 제공
- 키워드 필터 제공
- 메인 홈에서 그리드/리스트 토글 제공
- 폴더 컨텍스트에서는 그리드 중심 탐색 제공

### Tutorial

- 첫 방문 1회 노출
- skip, next, done 제공
- localStorage로 완료 여부 저장
- 다시 열 수 있는 진입점 제공

### Folder

- 폴더 생성 가능
- 생성 즉시 캐시 갱신
- 중복 이름 검증

### Problem Detail

- 문제 본문, 키워드, 폴더, 해설 이미지 제공
- 삭제 가능
- 컨텍스트 기반 back navigation 유지

### Registration

- 입력 validation
- objectUrl preview
- 성공 시 홈으로 복귀

## Non-Functional Requirements

- WebView 환경에서 과도한 hydrate를 피한다.
- 서버 prefetch 후 CSR hydration 구조를 유지한다.
- 로딩 중 skeleton 또는 loading route를 제공한다.
- 토큰/DS 클래스 우선 스타일링을 유지한다.

## Success Criteria

- 홈에서 전체 탐색이 가능하다.
- 폴더 클릭 시 해당 폴더 문제만 보인다.
- 키워드 상세는 컨텍스트를 잃지 않는다.
- 등록/삭제 후 query cache가 일관되게 갱신된다.
